# Project Overrides - LMS

## Цель
Фиксировать особенности backend LMS (FastAPI + PostgreSQL).

## Доменные ограничения
- Соблюдать слоистую архитектуру (`api` -> `services` -> `repos`).
- При изменении API обновлять соответствующую документацию.
- Учитывать бизнес-логику, реализованную триггерами БД.

## Ограничения по БД/интеграциям
- Изменения схемы только через Alembic миграции.
- По умолчанию запросы к БД через MCP в read-only режиме для анализа.
- Прямые write-запросы к БД только если это явно требует задача.

## Обязательные проверки перед интеграцией в `main`/`master`
- Пройти релевантные тесты (`pytest tests/` или целевой набор).
- Пройти smoke-check ключевых endpoint (health + затронутый API).
- Проверить миграции: применяются и не ломают базовые сценарии.
- Получить `review-gate: PASS`.

## Ограничения по безопасностным действиям
- Не логировать секреты (`DATABASE_URL`, API keys).
- Не коммитить `.env`/секреты/локальные конфиги доступа.
- Не выполнять destructive SQL без отдельного подтвержденного запроса.

## Исключения и особые кейсы
- Для срочных hotfix допускается сокращенный набор проверок, но `review-gate` перед интеграцией в `main`/`master` обязателен.
