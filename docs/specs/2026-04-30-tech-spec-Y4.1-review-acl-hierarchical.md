## 1. Цель

Закрыть архитектурный gap: преподаватель, привязанный к **корневому** курсу через `teacher_courses` (как требует DB-триггер), должен автоматически видеть pending review-заявки и help-requests для **всех потомков** этого курса в иерархии `course_parents` — без выдачи широкой роли `methodist`.

**Пользовательский результат:** в production можно ограничивать teacher'ов конкретной веткой курсов через привязку к одному root-курсу. Текущий обход через `methodist`-роль остаётся доступным как escape hatch (без изменений).

## 2. Контекст

### Текущее поведение (gap)

В `app/services/teacher_queue_service.py:53-64` определены два SQL-фильтра:

```python
HELP_REQUESTS_ACL_SQL = """
    (hr.assigned_teacher_id = :teacher_id
     OR EXISTS (SELECT 1 FROM student_teacher_links stl WHERE stl.student_id = hr.student_id AND stl.teacher_id = :teacher_id)
     OR (hr.course_id IS NOT NULL AND EXISTS (SELECT 1 FROM teacher_courses tc WHERE tc.course_id = hr.course_id AND tc.teacher_id = :teacher_id))
     OR EXISTS (SELECT 1 FROM user_roles ur JOIN roles r ON r.id = ur.role_id WHERE ur.user_id = :teacher_id AND r.name = 'methodist'))
"""

REVIEW_ACL_SQL = """
    (EXISTS (SELECT 1 FROM teacher_courses tc WHERE tc.course_id = t.course_id AND tc.teacher_id = :teacher_id)
     OR EXISTS (SELECT 1 FROM user_roles ur JOIN roles r ON r.id = ur.role_id WHERE ur.user_id = :teacher_id AND r.name = 'methodist'))
"""
```

Точное равенство `tc.course_id = <target>.course_id` — без рекурсии по `course_parents`. Все 380 SA_COM-задач лежат на non-root курсах; teacher, привязанный к root, не видит ни одну.

Дублирование: третий аналогичный фрагмент в `app/services/help_requests_service.py:255-257` (`_check_user_in_teacher_course`) + копия `HELP_REQUESTS_ACL_SQL` в `help_requests_service.py:312-316`.

## 7. Шаги реализации

### 7.1. Создание общего SQL-helper'а

**Исполнитель:** `/executor-pro` (security/contracts critical — общий ACL для review + help-requests)
**Ревью:** `/techlead-code-reviewer` (RBAC, recursive correctness)

**Файл:** `app/services/teacher_queue_service.py` — добавить module-level константу **выше** `HELP_REQUESTS_ACL_SQL`:

```python
# Y-4.1: рекурсивный ACL teacher_courses через course_parents.
# Teacher, привязанный к root-курсу, видит сущности всех потомков.
# Параметры bind: :teacher_id (int), :target_course_id_expr (column reference,
# подставляется через .format() в caller — НЕ из user-input).
#
# WITH RECURSIVE: глубина дерева 2 (max), безопасно без cycle protection
# (course_parents — DAG by design, см. M2 + триггер 20260127).
TEACHER_COURSE_HIERARCHY_ACL_TEMPLATE = """
    EXISTS (
        WITH RECURSIVE ancestor_chain AS (
            SELECT {target_course_col} AS course_id
            UNION ALL
            SELECT cp.parent_course_id
            FROM course_parents cp
            JOIN ancestor_chain a ON a.course_id = cp.course_id
        )
        SELECT 1 FROM teacher_courses tc
        WHERE tc.teacher_id = :teacher_id
          AND tc.course_id IN (SELECT course_id FROM ancestor_chain)
    )
"""


def teacher_course_acl(target_course_col: str) -> str:
    """Вернуть SQL-фрагмент `EXISTS (...)` для проверки доступа teacher_id к
    target_course_col (column reference, например `t.course_id` или `hr.course_id`).

    target_course_col — литерал из закрытого набора call-sites; user-input не
    попадает (вызывающий код ВСЕГДА передаёт static string).
    """
    return TEACHER_COURSE_HIERARCHY_ACL_TEMPLATE.format(
        target_course_col=target_course_col
    )  # nosec B608 — target_course_col из закрытого набора литералов
```

**Логика рекурсии:**
1. Стартовая строка `ancestor_chain` — сам target course (включая non-root начальную точку)
2. Рекурсия идёт **вверх** по `course_parents` (от child к parent), пока есть записи
3. Финальная проверка: `teacher_courses.course_id IN ancestor_chain` — teacher привязан к любому ancestor (включая сам target)

**Корректность:** для root-курса `ancestor_chain = {root}` — поведение идентично текущему точному равенству. Для child-курса `ancestor_chain = {child, parent, root, ...}` — teacher на root найдётся.
