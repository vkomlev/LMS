**Механизм дефекта (подтверждён кодом, не гипотеза):**
- `app/api/v1/attempts.py:448` — `solution_rules = SolutionRules.model_validate(task.solution_rules or {})`.

- Для случая «`short_answer` есть, но `accepted_answers` пуст» (`checking_service.py:_check_short_answer`) — цикл по `accepted_answers` пуст → `base_score=0` → `is_correct=False` всегда.

**Почему латентно:** по КомпЕГЭ/Поляков/РешуЕГЭ/Яндекс/Крылов — 0 результатов учеников (`task_results`). Всего по скоупу ученики трогали лишь 49 заданий. Сломается при первом же реальном использовании этих заданий.
