# tsk-369 — задания требуют файл-приложение, а файла нет

Дата: 2026-07-22 · База: прод `learn` (5.42.107.253) · Задача: [[tsk-369]] (P0)

## Что было

Задания ЕГЭ №3, 9, 10, 17, 18, 22, 24, 26, 27 формулируются коротко («Откройте файл
электронной таблицы…», «Текстовый файл состоит из…»), а всё различие между задачами одного
типа — в приложенном файле. Ученик открывал такое задание и видел текст без данных: решить
его невозможно в принципе. Наружу это не всплывало ничем — у задания есть и условие, и
правило проверки, формально всё исправно (в отличие от класса tsk-361, где правило пустое).

## Две поправки к исходной постановке

1. **`task_content.media` тут ни при чём.** Поле пусто у ВСЕХ 6303 активных заданий (`[]`
   либо отсутствует) — механизм не использует ни один импорт. Счёт «125 рабочих / 108
   сломанных» из постановки получился из-за того, что `'[]'` не попал в фильтр
   `IN ('', 'null')`. Рабочих по этому признаку не было вовсе.

2. **Мягкие переносы.** sdamgia расставляет U+00AD внутри слов («Тек­сто­вый файл
   со­сто­ит»), и поиск по условию терял целую партию. После их снятия картина другая:

| | заданий |
|---|---|
| активных заданий с файловым условием | 339 |
| из них файл был | 116 |
| **файла не было** | **223** |

Файл виден ученику только ссылкой `/api/v1/media/...` внутри `stem`: клиент SPW рисует
условие как HTML и поле `attached_file_paths` не читает (проверено по коду SPW). Значит и
чинить надо ссылкой в условии — метаданные ставятся заодно, для машинного учёта.

## Что сделано

**Привязано 220 заданий, 199 файлов (410 МБ).** Осталось 3 (список ниже).

| источник | заданий | как получен файл |
|---|---|---|
| kompege | 144 | `GET /api/v1/task/<id>` → `files[].url` |
| sdamgia | 35 | страница задачи → `/get_file?id=N` |
| yandex | 34 | `POST /api/v5/gpttr` → `markup.resources[].link` |
| близнец в LMS (Крылов) | 6 | готовый `sha_ext` задания `crylov:v<В>t<З>` |
| локальный архив книги Крылова | 1 | `ZADANIE-9/9var03.ods` |

По курсам: 151 → 63, 153 → 43, 160 → 35, 149 → 26, 145 → 19, 138 → 16, 141 → 15, прочие → 3.

Файлы уложены штатным помощником ContentBackbone `store_bytes_to_cas` (тот же CAS-путь,
тот же прод-бакет, идемпотентно по содержимому) и **каждый** проверен запросом к боевому
эндпоинту `GET /api/v1/media/<sha_ext>` → 200. Запись в БД — одной транзакцией, с бэкапом
прежних условий и построчной проверкой после COMMIT.

## Гейт: почему файл считается «тем самым»

ID сам по себе не доказательство (обжиг tsk-362: задание 3108 ссылалось на sdamgia:68243,
а по тому ID лежит другая задача). Поэтому:

1. **Базовый гейт** — дословный фрагмент 60 букв из середины условия LMS есть в тексте
   источника, И все значимые числа источника (3+ цифр) присутствуют в условии LMS.
2. **Второй гейт для `weak`** (сошёлся один признак из двух) — верный ответ LMS совпадает
   с ответом источника ИЛИ доля общего текста ≥ 0.75. Через него прошли 10 заданий: у
   части импорт обрезал таблицу-пример (её числа «пропали»), у части KaTeX рвал текст.
3. **Тип файла против формулировки** — «файл электронной таблицы» → xls/ods/xlsx/csv,
   «текстовый файл» → txt/odt. Расхождение = стоп.

**Независимая массовая проверка.** У 188 пар «задание ↔ источник» верный ответ есть с обеих
сторон: **184 совпали точно**, 3 — с точностью до формы записи («2; 6; 2; 3» = «2623»),
1 разошлась — у задания 3791 в LMS записан мусорный ответ «ы» (см. остаток).

**Три задачи решены по самому файлу — ответ сошёлся:**

| задание | что считали | получилось | ожидали |
|---|---|---|---|
| 3573 | минимальный отрезок, где `E` ровно 100 раз | 353 | 353 |
| 3000 | максимальный отрезок, где `K9` ≤ 150 раз и конец не `D` | 246960 | 246960 |
| 2837 | последняя строка с двумя парами повторов, сумма её чисел | 331 | 331 |

Последняя — файл из локального архива книги Крылова (варианта 3 нет онлайн), и решение по
файлу здесь единственный доступный гейт.

## Постоянный чек

`scripts/check_missing_attachments.py` + еженедельная обёртка
`scripts/check_missing_attachments_weekly.ps1` (пара к `check_ungradable_tasks_weekly.ps1`).
Ищет активные задания, где условие требует файл, а ссылки `/api/v1/media/` в условии нет.
Отдельно подсвечивает случай «файл есть в метаданных, но ученику не виден» — он чинится
дописыванием ссылки. Read-only, коды выхода 0/1/2.

Прогон после правки: **было 223 → стало 3.**

## Риски и хвосты

* **Формат ссылки** одинаков для всех 220 заданий (`<p><strong>Файл к заданию:</strong> …`
  в начале условия). Партия Крылова из tsk-317 использует свой вид (`<p><a>Файл к
  заданию</a></p>`) — визуально они рядом, но не идентичны; при желании привести к одному
  виду — отдельная мелкая задача.
* **Яндекс отдаёт один и тот же набор данных в 2–4 форматах** (csv/ods/xls/xlsx) — все
  привязаны, ученик выбирает удобный. Это увеличило объём хранилища, но не число сущностей.
* **Во время работы параллельная сессия** деактивировала задания 3332 и 3455 и завела
  взамен 9567 — это учтено (9567 получил файл, те два выбыли из выборки).
* `attached_file_paths` теперь заполнен у 220 заданий, но клиентом не используется. Если
  когда-нибудь SPW начнёт рисовать вложения из метаданных — ссылка в условии станет
  дублем, и её нужно будет убрать одним проходом.
