# Единый вид подписей заданий ЕГЭ — отчёт

**Дата:** 2026-06-04
**Задача:** tsk-107
**БД:** `Learn`, схема `public`

## Проблема

В списках заданий ЕГЭ часть пунктов отображалась как «Задача #N», часть — без
названия (пусто).

## Корневая причина

Фронт SPW формирует подпись так
([use-course-syllabus.ts:283](../../spw/lib/learning/use-course-syllabus.ts)):

```ts
title: tc.title ?? `Задача #${raw.task_id}`
```

Оператор `??` подставляет автоподпись только при `null`/`undefined`. Поле
`tasks.task_content.title` в БД заполнено двумя разными способами в зависимости
от пути импорта:

| `title` в БД | Кол-во | Источник | UI |
|---|---|---|---|
| JSON `null` | 1753 | старый импорт | «Задача #N» |
| пустая строка `""` | 1080 | D4-конвейер CB (kompege/yandex/polyakov/sdamgia) | пусто |

Пустую строку `??` не перехватывает → задание без подписи. Осмысленных названий
ни у одного из 2834 заданий нет — подпись всегда генерится фронтом.

## Решение (LMS-only, по выбору оператора)

1. **Нормализация данных.** 1080 заданий: `task_content.title` `""` → JSON `null`
   (`jsonb_set`, одна транзакция, флаг `app.skip_task_order_trigger`). После:
   0 пустых строк, 2833 `null` (+1 без ключа). Все дают «Задача #N».
2. **Валидатор схемы** (`app/schemas/task_content.py`): `field_validator("title",
   mode="before")` приводит пустую/пробельную строку к `None`. Закрывает повтор
   при будущих импортах независимо от того, что присылает ContentBackbone.
3. **Тесты** `tests/test_task_title_normalize.py` (7 шт., все зелёные).

## Верификация

- БД: `empty_str=0, json_null=2833, non_empty=0, total=2834`.
- Валидатор: `""`/пробелы → `None`, реальное имя сохраняется, `None`/отсутствие → `None`.
- `pytest -k task_content` — 16 passed; новый модуль — 7 passed.

## Артефакты

- Скрипт нормализации: `reviews/evidence/2026-06-04-task-title-normalize.py`
- Схема: `app/schemas/task_content.py` (валидатор `_empty_title_to_none`)
- Тесты: `tests/test_task_title_normalize.py`

## Risks / Follow-ups

- **Не делалось** (вне выбранного объёма): починка фронта SPW (`??` → обработка
  пустой строки) и источника в ContentBackbone (перестать писать `""`). После
  валидатора в LMS это не критично — пустые строки больше не сохранятся. При
  желании — отдельные задачи в SPW/CB.
- Осмысленные названия заданий вместо «Задача #N» — отдельная контентная задача.
- Схема `task_content` — публичный контракт импорта; изменение мягкое (приведение
  ""→None, обратная совместимость сохранена), миграции/endpoint не затронуты.
