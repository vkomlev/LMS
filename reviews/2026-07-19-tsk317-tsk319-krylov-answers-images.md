## Контекст
36 активных Крылов-заданий ЕГЭ в LMS (`external_uid LIKE 'crylov:%'`) не имели
`solution_rules`, `answer_raw` и картинок — приём ответа падал бы на первой же
попытке (общий дефект аудита tsk-299, F3/F6). Оператор сузил скоуп: варианты
1/5/11/16 полностью + несколько из 2-3 (наличие разбора в ТГ/CB).

**Investigate-first показал:** все 36 существующих LMS-заданий Крылова — это
ровно варианты 1/5/11/16 (12+9+7+8 задач), без остатка. Кандидатов из
вариантов 2-3 в LMS нет вовсе — это отдельная развилка, зафиксирована в
tsk-319 (см. ниже).

## Источник ответов — НЕ ContentBackbone (важная находка)
CB `external_tasks.task` (`source='pdf:d4:pdf:crylov:vN:20260602'`) содержит
автоматически распознанные ответы из таблицы «ОТВЕТЫ» той же книги (страницы
245-249), извлечённые table-OCR конвейером CB. Проверка показала: OCR для
вариантов 5/11/16 массово испорчен — значения вида `or`, `ae`, `sooo`, `ao`,
`aooo` не совпадают с реальными числами (совпадает только колонка варианта 1,
самая близкая к левому краю таблицы). `answer_extraction_meta.confidence=0.85`
одинаков у ВСЕХ ячеек (в т.ч. заведомо неверных) — не индикатор качества.

**Реальный источник:** прямое чтение отрендерённых страниц исходного PDF
(`ContentBackbone/tests/fixtures/external_tasks/pdf/Крылов. ЕГЭ по информатике
2026.pdf`, 243 стр., только сканы — текстового слоя нет). Раздел «ОТВЕТЫ» —
физические страницы 239-242 (печатные 245-248), рендер 200-350 DPI через
`pdftoppm`, прочитан визуально построчно по таблице (вариант × номер задания).

**Кросс-проверка (3 независимых источника, 0 расхождений):**
1. future-step.ru/task/1618/ (внешний сайт с разборами) — независимо подтвердил
   v1t16=6.
2. Уцелевшие (не искажённые OCR) фрагменты CB-извлечения — совпали: v16t24=10007,
   v1-колонка целиком (18/18 сверенных значений).
3. Прямое чтение исходного PDF (основной источник для всех 36).

UPDATE — только по явному списку 36 id, только пока `solution_rules IS NULL`
(WHERE-guard) — повторный запуск ничего не изменит и не упадёт молча (явно

(`task_results`) — низкий риск. Обратимо (`solution_rules → null`, `<img>`
строка убирается по тому же анкеру).
