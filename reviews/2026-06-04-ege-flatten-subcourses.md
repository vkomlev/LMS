# Плоская иерархия подкурсов курса ЕГЭ — отчёт

**Дата:** 2026-06-04
**Задача:** tsk-106
**БД:** `Learn`, схема `public` (PostgreSQL, localhost)
**Тип:** правка данных в одной транзакции (не миграция схемы)

## Контекст

Курс ЕГЭ (id 112) имел двухуровневую иерархию подкурсов: «Навигатор по заданию N»
(id 113-137) → внутри «Задание N ЕГЭ…». Навигаторы дублировали структуру.
Требовалось слить навигатор с заданием (имя — от задания), поднять задания на
верхний уровень, сохранить материалы обоих, оставить подкурсы-«довески», и
исправить ошибочное вложение «Задания 9» под «Навигатор 12».

## Что сделано

1. **25 слияний навигатор → задание.** Материалы навигатора перенесены в курс-задание
   (идут первыми по порядку), навигатор удалён, задание поднято прямым ребёнком
   корня 112 с сохранением порядкового номера навигатора (order 2..26).
2. **Довески сохранены как подкурсы слитого задания:** 165 «Черепашья графика» →
   под 157 (Задание 6); 161 «Формулы и функции в Excel», 164 «Основы работы в
   электронных таблицах» → под 160 (Задание 9).
3. **Ошибка 9/12 исправлена:** курс 160 «Задание 9» возвращён из-под Навигатора 12
   (124) на верхний уровень (слот навигатора 9, order 10) и слит с навигатором 121.

## DB Findings (разведка до правок)

- Иерархия — `course_parents` (PK `course_id, parent_course_id`), один родитель на курс.
- FK: `materials/tasks/student_course_state/user_courses/teacher_courses` —
  `ON DELETE CASCADE` (удаление навигатора уничтожило бы материалы → перенос ДО удаления);
  `social_posts` — `NO ACTION`; `attempts/help_requests` — `SET NULL`.
- Зависимых данных нет: 0 записей в user_courses, teacher_courses,
  student_course_state, social_posts, course_dependencies, attempts,
  student_material_progress по навигаторам.
- `order_position` материалов — без уникального ограничения; `UNIQUE(course_id, external_uid)` —
  коллизий между навигатором и заданием нет, NULL external_uid отсутствуют.

## Триггеры (учтены при правке)

- `trg_set_material_order_position` (BEFORE INS/UPD) — каскадная нормализация порядка
  материалов; отключён на транзакцию флагом `app.skip_material_order_trigger='true'`,
  порядок нормализован вручную (материалы навигатора первыми, 0-based, непрерывно).
- `trg_set_course_parent_order_number` (BEFORE INS/UPD) — отключён флагом
  `app.skip_course_parent_order_trigger='true'`, order_number выставлен явно.
- `trg_reorder_course_parents_after_delete` (AFTER DELETE, без флага) — конфликтовал
  при каскадном удалении 25 строк-сестёр; временно отключён через
  `ALTER TABLE … DISABLE/ENABLE TRIGGER` внутри транзакции.

## Верификация (до COMMIT)

- навигаторов осталось: 0 (ожид. 0)
- прямых детей корня 112: 25 (ожид. 25)
- удалено курсов: 25 (ожид. 25)
- материалов в наборе до/после: 267/267 (совпало)
- довески: 165→157, 161→160, 164→160 (как ожидалось)

Независимая перепроверка через MCP подтвердила итоговое дерево: 25 заданий
по порядку, суммы материалов сошлись (напр. Задание 1: 8+4=12, Задание 9: 3+8=11,
Задание 6: 9+3=12), подкурсы на местах.
