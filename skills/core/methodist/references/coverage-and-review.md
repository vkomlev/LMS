# Полнота покрытия и двойная проверка плана

Источник истины для skill `methodist`. Подключается в Шаге 0. Защищает от
поверхностных планов: гейт полноты против инвентаря источника + самопроверка
«двумя глазами» (ученик / сеньор) перед выдачей.

## Часть 1: Гейт полноты против инвентаря источника

Для запроса уровня **трека / куррикулума** (а не одного модуля) план обязан
покрыть весь инвентарь курса, а не только «модуль по теме».

**Алгоритм (Шаг 0 / Шаг 2):**
1. Glob `docs/v2/*.md` — получить полный список файлов.
2. Для **каждого** файла зафиксировать решение: `включён в план` / `исключён`
   с причиной (например: «X3 = Трек 2+3, к Треку 1 не относится»).
3. Сверить результат с обязательным-по-стадии минимумом ниже. Пропуск
   обязательного = план не готов, доработать.
4. В артефакт включить раздел **«Карта покрытия источника»**: таблица файл →
   включён/исключён → где в плане / причина исключения.

### Обязательный минимум по стадии

| Материал (тип) | новичок | с-базой | уверенный |
|---|---|---|---|
| Онбординг: манифест, карта экосистемы, квиз, «как проходить» | обязательно | обязательно | кратко |
| Базовые понятия / глоссарий (`F-Глоссарий`) | **обязательно, в начале** | обзорно | по запросу |
| Полный Фундамент Claude (все `F-Claude*` файлы) — отдельная фаза, не сжимать | **обязательно** | ускоренно | ускоренно |
| Техбаза (Ф0–Ф6) | по необходимости, глоссарий первым | опционально | пропуск |
| Бесплатные альтернативы (`F-Бесплатные-альтернативы`) | **обязательно** (cost-блокер) | обязательно | обязательно |
| Безопасность (`F-Безопасность`) | обязательно | обязательно | обязательно |
| Промпт-библиотека (`F-Промпт-библиотека`) | как опора в каждом паттерне | да | да |
| Кросс-трековые модули (`X1*`, `X2*` с участием Трека 1) | показать как мост (опционально) | предложить | предложить |
| Расходы/оптимизация Claude (`F-Claude-расходы*`) | обязательно | обязательно | обязательно |

**Принцип для новичка:** ни «базовые понятия», ни «бесплатные альтернативы», ни
полный фундамент НЕ срезаются ради компактности. Срез фундамента для нулевого
ученика = блокер прохождения, не оптимизация.

**Кросс-модули:** `X*`-файлы, где участвует Трек 1, минимум упомянуть как мост к
другим трекам в финале; не выкидывать молча.

### Курируемая выборка ContentAnalyzer

Если пользователь дал путь/бриф `ca-curated`, `curate.py` или
`output/distribution/courses/...`, использовать эту выборку как вторичный
материал для сверки покрытия, примеров и идей заданий.

- `docs/v2` остаётся источником истины; CA-выборка не заменяет инвентарь
  источника и не может отменить обязательные материалы.
- CA-выборка не входит в обязательный STOP-гейт Шага 0 и не читается при
  обычном проектировании модуля без явного входа.
- При расхождении выборки с `docs/v2` зафиксировать расхождение в карте
  покрытия и опираться на `docs/v2`, если оператор явно не изменил источник.

## Часть 2: Двойная проверка (гейт перед верификацией)

После черновика плана/заданий — прогнать **двумя взглядами**. Это обязательный
шаг, не опция. Любой ответ «нет/не уверен» = доработать до повторного прохода.

### Глазами ученика (целевая ЦА × стадия)

Поставить себя на место конкретной стадии (для новичка — 0–1 год, без
коммерческого опыта) и честно ответить:

**Механическая трассировка предпосылок (обязательная процедура, не «на глаз»):**
Для **каждого** шага S:
1. Выписать список понятий / терминов / инструментов, без которых S не выполнить
   (например, для «посчитать токены и стоимость API» это: токен(AI), модель,
   Claude API, промпт).
2. Для каждого пункта списка найти **более ранний** шаг, где он **явно разобран**
   (учебный/глоссарный шаг, стоящий по порядку ДО S).
3. Если хоть один требуемый пункт разбирается **позже S**, **параллельно S** или
   формулировкой «по мере встречи / по необходимости» — это **дефект порядка**.
   Переставить разбор понятия в шаг строго до S. «Параллельно» и «по ходу» не
   считается «разобрано раньше».

> Инвариант: служебный/инфраструктурный шаг (счёт токенов, калькулятор расходов,
> запрос к API, деплой) НЕ может идти раньше шага, где явно разобраны понятия,
> которые он использует. Базовые понятия — раньше инструмента, который их требует.

- [ ] Механическая трассировка пройдена: ни одного понятия, требуемого шагом,
      не разобрано позже/параллельно этого шага
- [ ] Я физически смогу выполнить **каждый** шаг с теми предпосылками, что уже
      даны ранее в плане? (нет скрытого «и тут просто сделай X», где X не учили)
- [ ] Нет ли скачка сложности между соседними шагами (перегруз)?
- [ ] Нет ли шагов слишком тривиальных для заявленной стадии (недогруз, скука)?
- [ ] Хватает ли заложенного времени реально, а не по оптимистичной оценке?
- [ ] Понятно ли, что именно я предъявляю на каждом шаге (критерий не размыт)?
- [ ] Для классификаций и пар родственных терминов есть критерий различия: как
      проверить новый пример, а не только узнать пример из текста?
- [ ] Миссии безопасны: не требуют настоящих паролей, токенов, аккаунтов, ПД и
      не противоречат правилам самого урока?
- [ ] Если я застряну — план говорит, куда идти (опора, цепочка AI-наставника)?

### Глазами сеньора по теме

Поставить себя на место практикующего эксперта именно по этой теме (не общего
методиста) и проверить:

- [ ] Прогрессия логически последовательна, без дыр (каждый шаг следует из
      предыдущего, ничего не «телепортируется»)?
- [ ] Все ключевые вопросы темы закрыты (нет «а про это забыли»)?
- [ ] Дан **полный набор инструментов/материалов** под цель — ничего
      обязательного не выпало (инструмент, понятие, источник)?
- [ ] Нет поверхностности: каждый учебный результат подкреплён конкретным
      файлом `docs/v2`, а не общими словами?
- [ ] Порядок тем соответствует тому, как это осваивают на практике, а не
      «как удобно записать»?
- [ ] Нет ли устаревших/неактуальных для текущего стека вещей?

### Линза освоения (mastery) — финальная, по ЗАДАНИЯМ темы

После двух взглядов — последний проход именно по ЗАДАНИЯМ темы (не по плану).
Цель скилла — чтобы ученик НАУЧИЛСЯ ДЕЛАТЬ тему сам, а не чтобы тема была
формально «покрыта материалом».

- [ ] **Сможет ли ученик САМ сделать эту тему после ЭТИХ заданий** (а не просто
      прочитать материал)?
- [ ] **Каждый ли изученный приём/метод он где-то применил руками** (срезы,
      `count`/`find`/`replace` и т.п. не остались «только в материале»)? Сверить с
      гейтом покрытия приёмов (assignment-rules §3.4).
- [ ] **Хватает ли объёма и РАЗНООБРАЗИЯ** (не 3 однотипных задания на
      содержательную тему; микс типов — assignment-rules §8 п.8)?
- [ ] **Рост по профилю ЦА** (difficulty-and-design, «Профиль прогрессии»): объём и
      сложность заданий растут от старта к финалу в коридоре профиля; для не-детской
      ЦА к концу есть многострочные программы и ≥ требуемого числа проектных
      (assignment-rules §3.5)? «Детский» одношаговый формат весь курс при
      предвузовской/взрослой ЦА = брак.

Любой «нет» = задания дают узнавание, а не умение → доработать задания (не план),
не выдавать.

### Анти-поверхностность (общий критерий)

- [ ] Каждый пункт плана трассируется к источнику (файл docs/v2 в `docs-refs`).
- [ ] Нет формулировок-заглушек («разобрать основы», «познакомиться с темой»)
      без конкретного результата-действия и источника.
- [ ] Глубина соответствует заявленному уровню детализации (карта-план — связи и
      результаты; модуль — задания с критериями).
- [ ] **Конверт урока** (difficulty-and-design Часть 3 п.7): у КАЖДОГО урока цель-действие
      на входе («после урока сможешь …») и итог на выходе («Итог: …»); для спиральных —
      мостик «Вспомни: …»; первый программирующий урок — подсказка запуска. Урок без
      цели/итога = незавершён.
- [ ] **Визуал на абстрактной теме:** темы без наглядного объекта (логика/таблицы
      истинности, системы счисления, структуры данных, алгоритмы, функции) помечены как
      требующие ASCII-схему ВНУТРИ материала, не только обложку (см. SKILL «Визуалы»).
      Абстрактная тема с одной обложкой = визуальный голод, брак.

**Вывод двойной проверки** включить в Контракт результата отдельным полем:
что проверено, что переделано после прохода, остаточные риски.
