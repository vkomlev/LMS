# Портрет целевой аудитории

## Принцип
Проверка проводится не «четырьмя персонами», а одной конкретной ЦА курса или
сегмента. Если ЦА меняется, меняется и сегмент проверки. Портрет фиксируется до
чтения первого урока и дальше служит фильтром: вопрос засчитывается только тогда,
когда он правдоподобен для этой аудитории.

## Сегментация
Новый сегмент нужен, если меняется хотя бы один пункт:
- возраст или класс;
- школьная ступень или программа;
- набор предпосылок;
- цифровой опыт;
- сложность материала;
- формат обучения и мотивация.

Примеры:
- курс для одной группы 11-14 лет -> один сегмент;
- информатика 5-11 -> отдельные сегменты по классам или устойчивым возрастным
  блокам;
- курс, где первые модули для новичков, а последние для уверенных учеников ->
  минимум два сегмента.

## Пессимистичная калибровка
Портрет строится по нижнему квартилю ЦА: слабый или средний ученик с пробелами,
но не карикатурный персонаж. Цель — поймать реальные вопросы, а не доказать, что
курс непроходим.

Формула: «типичный ученик этой ЦА в неблагоприятном варианте».

Для школьного курса это может быть троечник, который:
- освоил прошлую программу неглубоко;
- помнит отдельные слова, но путается в абстракциях;
- лучше понимает через пример, картинку и короткое действие;
- не держит длинную цепочку рассуждений без опор.

## Источники портрета
Использовать только доступные факты:
- план курса и README;
- описание ЦА в исходниках;
- предыдущие разделы курса;
- школьный класс и возраст;
- требования программы, если они есть в материалах;
- сведения оператора.

Если критически важной информации нет, задать один короткий вопрос оператору.
Не выдумывать профиль ради красивого отчёта.

## Шаблон `audience.md`

```markdown
# Портрет ЦА: <курс / сегмент>

Источник портрета: <README / plan / оператор / вывод из структуры курса>
Сегмент: <глава, класс, раздел, диапазон уроков>
Дата: <YYYY-MM-DD>

## Кто это
- Возраст / класс:
- Учебный контекст:
- Успеваемость, пессимистичная калибровка:
- Мотивация:

## Что уже умеет
- ...

## Что скорее всего не умеет
- ...

## Что мог забыть
- ...

## Словарь и абстракции
- Простые слова:
- Термины на грани:
- Слова, которые требуют примера:

## Порог внимания и формат
- Сколько выдерживает сплошного текста:
- Какие опоры помогают:
- Какие форматы перегружают:

## Цифровой опыт
- ...

## Фильтр вопросов
Вопрос считается валидным, если <короткое правило для этой ЦА>.
Вопрос не считается валидным, если <что эта ЦА уже точно знает>.
```

## Правила применения портрета
- Не переносить портрет одного сегмента на другой.
- Не проверять курс взрослым, если курс сделан для ребёнка, и наоборот.
- Не повышать уровень ученика, чтобы «помочь автору». Если курс не дал понятие,
  ученик его не знает.
- Не понижать уровень до абсурда. Если ЦА точно умеет читать простые инструкции,
  не задавать вопросы уровня «что такое точка».
