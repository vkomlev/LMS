# Codex Skill Standard

Codex-adapted version of the Claude skill standard. Keep the useful structure, but avoid Claude-only `allowed-tools` assumptions.

## Frontmatter

Required:

```yaml
---
name: skill-name
description: "What the skill does and when to use it."
---
```

Optional when useful:

- `version`
- `license`
- other Codex-supported metadata already used by system skills

Do not add Claude-only fields such as `allowed-tools` unless a target runtime explicitly supports them.

## Required Sections

Use concise English or Russian consistently within a skill:

1. Role or Purpose
2. When to Use or Trigger Rules
3. Workflow
4. Output Contract
5. Quality Rules

Complex checklists belong in `references/`; `SKILL.md` should route and constrain, not become a manual.

## Runtime Interface

For Codex UI visibility, add `agents/openai.yaml`:

```yaml
interface:
  display_name: "Skill Name"
  short_description: "Short user-facing summary"
  default_prompt: "Default prompt for this skill."
```

## Size And Maintenance

- Simple skills: 30-80 lines.
- Complex skills: keep `SKILL.md` compact and put detailed checks in references.
- Generated skills with an `AUTOGENERATED` guard must be changed via `Docs/ai-booster/skill-template-sources.json`.
- Every source edit should be rolled out through `scripts/package-skills.py` when the skill is managed.
