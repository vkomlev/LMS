# Codex Project Binding

Use this when importing Claude project setup patterns into Codex.

## Codex Runtime Files

- Global runtime root: `C:/Users/user/.codex`.
- Runtime skills: `C:/Users/user/.codex/skills/<skill>`.
- Global config: `C:/Users/user/.codex/config.toml`.
- Project mirrors: `<project>/skills/core/<skill>`.
- Project discovery file: `<project>/AGENTS.md`.

## Mapping From Claude Concepts

| Claude concept | Codex equivalent | Notes |
|---|---|---|
| `~/.claude/skills/<skill>` | `~/.codex/skills/<skill>` | Runtime mirror, not always source-of-truth. |
| global `CLAUDE.md` | global Codex instructions / generated project `AGENTS.md` | Keep path casing and platform names accurate. |
| project `CLAUDE.md` | project `AGENTS.md` | Generated by `package-skills.py` from `skills/core`. |
| `.claude/settings.json` permissions | Codex tools, sandbox, approval policy, and config | Do not copy Claude allow/deny entries directly. |
| Claude MCP settings | `[mcp_servers.*]` in `config.toml` or app-provided MCP tools | Keep secrets out of committed files. |
| Claude hooks | Codex automations, task reminders, or explicit checklist rules | Map the intent, not the shell hook. |

## Project Trust And Discovery

For a project to be comfortable in Codex:

1. The project should be listed in `Docs/ai-booster/project-registry.md`.
2. If managed skills are needed, set `skills/core` to `yes`.
3. Run `package-skills.py --dry-run` before changing mirrors.
4. Run `package-skills.py --skills <name>` to sync source, runtime, project mirrors, and `AGENTS.md`.
5. If the Codex app does not trust the project, add a `[projects.'path'] trust_level = "trusted"` entry in `C:/Users/user/.codex/config.toml` only after confirming the path.

## MCP And Credentials

- Prefer environment variables or local uncommitted config for credentials.
- Validate MCP server names against active Codex tools before documenting a workflow.
- When adding DB access, update both the relevant domain skill and the project handoff docs.

## Verification

- `SKILL.md` frontmatter parses.
- `quick_validate.py` passes for runtime skill folders.
- Runtime and source hashes match for managed skills.
- Project `AGENTS.md` lists the expected skill after packaging.
- A sample prompt triggers the intended skill in a fresh Codex session.
