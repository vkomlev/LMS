# Правила обращения к оператору

Используй этот порядок во всех сеансах Codex. Краткая обязательная версия устанавливается в `C:/Users/user/.codex/AGENTS.md`.

## Основной принцип

По умолчанию агент действует самостоятельно. Обращение к оператору допустимо только когда:

1. Без действия оператора дальнейшая работа невозможна: нужны недоступные учетные данные, подтверждение входа, ручное действие во внешнем окне, физическое устройство или обязательное разрешение на необратимое действие.
2. Оператор прямо потребовал согласование перед действием.
3. Найдена развилка, которую нельзя разрешить по коду, документации или ранее принятым решениям, а выбор меняет цель или границы задачи.

Если заблокирован только один шаг, продолжай все независимые действия.

## Проверка перед обращением

Перед передачей шага оператору проверь:

1. Есть ли доступная команда или программный интерфейс вместо ручных действий.
2. Можно ли выполнить действие с текущими правами.
3. Можно ли получить ответ из файлов, журналов, базы данных, документации или браузера.
4. Можно ли выбрать безопасный обратимый вариант в рамках задачи.
5. Можно ли подготовить и проверить остальные части задачи без участия оператора.

## Формат инструкции

Когда действие оператора действительно необходимо, выдай:

```text
### Инструкция для оператора: <короткое название>

Цель: <что нужно получить>
Где выполнить: <программа, окно, адрес или устройство>
Шаги:
1. <точное действие> -> ожидаемый результат: <что должно появиться>
2. ...
Что вернуть агенту: <текст, снимок экрана, код ошибки или подтверждение>
Если возникла ошибка:
- <условие> -> <что сделать>
```

Не передавай оператору команды и проверки, которые можешь выполнить самостоятельно.
